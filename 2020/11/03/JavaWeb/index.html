JavaWeb

一、理论概念

请求方式：

get post head，delete， put，tract·····

get：能求能够携带的参数比较少，大小有限制会在浏览器的URL地址栏中显示，不安全，但高效

post：没有限制，url不会显示，安全，但不高效。

响应代码

200 请求响应成功

3xx 请求重定向

404 找不到资源

JSP

九大内置对象

  • PageContext 存东西
  • Request 存东西
  • Response
  • Session 存东西
  • Application [ ServletContext ] 存东西
  • config [ servletConfig ]
  • out
  • page [几乎不用]
  • exception

**request :**客户端向服务器发起请求，产生数据，用户看完就删 例如：新闻

session： 客户端向服务器发起请求，产生数据，用户用完一会还有用 例如：购物车

application ：客户端想武器发起请求，产生数据，一个用户用完，其他用户可能还会使用 例如：聊天数据
